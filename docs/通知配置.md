## 一、整体流程

```
飞书 / 企业微信群
        │ ① 创建机器人，得到 webhook URL
        ▼
GitHub 仓库 Secrets / Variables
        │ ② NOTIFY_TYPE + FEISHU_WEBHOOK_URL / WECOM_WEBHOOK_URL
        ▼
GitHub Actions 运行 main.py
        │ ③ utils/export_github_env.py 自动把上面两个值注入环境变量
        ▼
utils/notify.py
        │ ④ 任务完成 / 失败时调用 webhook
        ▼
你的群里收到消息
```

需要你做两件事：**①创建机器人拿到 webhook URL**，**②把 URL 填到 GitHub**。
第 ③ ④ 步代码已经全自动。

## 二、获取飞书 webhook

> **说明**：飞书的 webhook URL 形如 `https://open.feishu.cn/open-apis/bot/v2/hook/<32位字符串>`。
> 任何拿到这个 URL 的人都能往你群里发消息，**不要泄露**。

### 2.1 创建群机器人

1. 打开飞书 PC 端 / 移动端，**进入想要接收通知的群**（建议单独建一个群只给这一类机器人用）
2. 点群右上角设置图标 → **群机器人** → **添加机器人**
3. 在机器人列表里找到 **「自定义机器人」**（图标是个齿轮，不是飞书官方应用），点添加
4. 起个名字（如"续火花通知"）和描述，点"下一步"

### 2.2 选择安全设置

飞书要求至少开启一种安全策略，三选一：

| 选项 | 说明 | 推荐度 |
|------|------|--------|
| **关键词** | 消息正文必须包含指定关键词，否则被丢弃 | ⭐⭐⭐ 最简单 |
| **IP 白名单** | 限制只允许某些 IP 调用 | ❌ 不推荐，GitHub Actions 的 IP 是动态的 |
| **签名校验** | 每次请求带 HMAC-SHA256 签名 | ⭐⭐ 最安全，但需要改代码（本项目暂不支持） |

**推荐选「关键词」**，关键词填 **`续火花`**（必须和代码里的标题对上）。

> 本项目代码会发 "续火花完成" / "续火花运行完成" / "续火花运行失败" 等标题，
> 关键词设为 `续火花` 全部能命中。

如果你嫌麻烦，也可以**不开任何校验**——飞书会让你勾"我已阅读并同意"然后允许跳过。

### 2.3 复制 Webhook URL

点"完成"后，飞书会显示一段 URL：

```
https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

**完整复制下来**，等下要填到 GitHub。

## 三、获取企业微信 webhook

> ⚠️ **企业微信 ≠ 个人微信**。个人微信群没有官方的群机器人 webhook。
> 如果你只用个人微信，请用飞书方案。

> **说明**：企业微信的 webhook URL 形如 `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=<uuid>`。
> 同样不要泄露。

### 3.1 前置条件

- 你需要有一个企业微信账号（注册免费）
- 已经加入了一个企业（或者自己建一个，10 秒）
- 在企业微信内部创建了一个群（哪怕只有你一个人也可以）

### 3.2 创建群机器人

1. 打开企业微信 PC 端 / 移动端，进入要接收通知的群
2. 点群右上角的 **「…」** → **群机器人** → **添加机器人**
3. 起名字（如"续火花通知"），可以选个头像
4. 点确定

### 3.3 复制 Webhook URL

机器人创建后，点机器人头像可以看到 **Webhook 地址**：

```
https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

**完整复制**（含 `?key=` 参数），等下要填到 GitHub。

> 企业微信群机器人没有关键词 / 签名等安全设置，URL 本身就是凭证。

## 四、配置到 GitHub 仓库

打开 GitHub 仓库 → **Settings** → **Secrets and variables** → **Actions**。

页面顶部有两个 tab：**Variables**（变量，明文）和 **Secrets**（密钥，加密）。

### 4.1 添加 NOTIFY_TYPE（Variables）

切到 **Variables** tab → **New repository variable**：

| 字段 | 值 |
|------|----|
| Name | `NOTIFY_TYPE` |
| Value | 三选一：<br>• `feishu` — 只发飞书<br>• `wecom` — 只发企业微信<br>• `feishu,wecom` — 两个都发（注意是英文逗号，不要空格） |

> **为什么放 Variables 不放 Secrets？** 这个值不是机密，且明文有助于在 Actions 日志里快速看到当前配置。
>
> **不设这个变量等于关闭通知**——任务照常运行，只是不推送。

### 4.2 添加 webhook URL（Secrets）

切到 **Secrets** tab → **New repository secret**。**按你 NOTIFY_TYPE 选择的渠道添加对应的项**：

#### 4.2.1 添加 FEISHU_WEBHOOK_URL（如果你选了 feishu）

| 字段 | 值 |
|------|----|
| Name | `FEISHU_WEBHOOK_URL` |
| Secret | 第 2.3 步复制的完整 URL，例如 `https://open.feishu.cn/open-apis/bot/v2/hook/abcd1234-...` |

#### 4.2.2 添加 WECOM_WEBHOOK_URL（如果你选了 wecom）

| 字段 | 值 |
|------|----|
| Name | `WECOM_WEBHOOK_URL` |
| Secret | 第 3.3 步复制的完整 URL，**含 `?key=` 参数**，例如 `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=abcd1234-...` |

### 4.3 不需要改 workflow 文件

`utils/export_github_env.py` 已经会把仓库所有 `vars` 和 `secrets` 自动注入进 Python
进程的环境变量，所以**新增 secret / variable 不用改任何 .yml 或代码**。

> 如果你换了一个新 secret 名字（不是上面这两个标准名），代码不会读到。命名必须严格对上。

### 4.4 验证配置已生效

去 Actions 页面手动触发一次 workflow（**Run workflow** 按钮），等运行结束。

- 如果**收到消息** → 配置成功 ✅
- 如果**没收到** → 看下面 [七、常见故障排查](#七常见故障排查)

## 五、本地测试

如果你不想等 Actions 跑一次，可以本地直接测试 webhook：

### 5.1 飞书

```bash
export NOTIFY_TYPE=feishu
export FEISHU_WEBHOOK_URL='https://open.feishu.cn/open-apis/bot/v2/hook/<你的>'
python -m utils.notify
```

预期：

- 控制台打印 `已发送 feishu 通知`
- 飞书群里收到一条标题为"测试通知"的绿色卡片

### 5.2 企业微信

```bash
export NOTIFY_TYPE=wecom
export WECOM_WEBHOOK_URL='https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=<你的>'
python -m utils.notify
```

预期：

- 控制台打印 `已发送 wecom 通知`
- 企业微信群里收到一条 markdown 消息，开头是 ✅ 测试通知

### 5.3 两个一起

```bash
export NOTIFY_TYPE=feishu,wecom
export FEISHU_WEBHOOK_URL='...'
export WECOM_WEBHOOK_URL='...'
python -m utils.notify
```

### 5.4 用 curl 直接测（绕开 Python，验证 URL 本身有效）

**飞书：**

```bash
curl -X POST 'https://open.feishu.cn/open-apis/bot/v2/hook/<你的>' \
     -H 'Content-Type: application/json' \
     -d '{"msg_type":"text","content":{"text":"续火花 测试一下"}}'
```

预期返回 `{"code":0,"msg":"success",...}`

**企业微信：**

```bash
curl -X POST 'https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=<你的>' \
     -H 'Content-Type: application/json' \
     -d '{"msgtype":"text","text":{"content":"测试一下"}}'
```

预期返回 `{"errcode":0,"errmsg":"ok"}`

## 七、常见故障排查

### 7.1 没收到任何消息

按顺序检查：

1. **`NOTIFY_TYPE` 设了吗？** 没设 = 不发。在仓库 Settings → Variables 里确认。
2. **拼写正确吗？** 必须是 `feishu` / `wecom`，全小写，多渠道用英文逗号分隔。
3. **webhook secret 名字对吗？** 必须严格是 `FEISHU_WEBHOOK_URL` / `WECOM_WEBHOOK_URL`。
4. **去 Actions 日志里搜关键字**：
   - `已发送 feishu 通知` / `已发送 wecom 通知` → 发送成功（消息被 webhook 接住了，但群里没显示通常是关键词没命中）
   - `渠道 feishu 未配置 FEISHU_WEBHOOK_URL` → secret 没设或名字错了
   - `未知通知渠道` → NOTIFY_TYPE 拼错了
   - `发送 feishu 通知失败` → 看后面跟的错误信息

### 7.2 飞书：看到 "已发送" 但群里没消息

99% 是**关键词没命中**。

- 你创建机器人时设的关键词是什么？
- 本项目发送的标题包含 `续火花` 三字，关键词必须**完全等于**或**是这三字的子串**，否则被静默丢弃。
- 修正方法：进飞书群机器人设置，把关键词改成 `续火花`。

### 7.3 飞书：返回 `19001`

```
飞书返回非 0：{'code': 19001, 'msg': '...'}
```

- 关键词必填但消息中不含。同 7.2 处理。

### 7.4 飞书：返回 `19024` / `19021`

- 你启用了"签名校验"，但本项目没传签名。请改成"关键词"或不开校验。

### 7.5 企业微信：返回 `93000`

```
企业微信返回非 0：{'errcode': 93000, ...}
```

- webhook URL 错了。最常见的原因：复制时漏了 `?key=...` 参数。重新复制完整 URL。

### 7.6 企业微信：返回 `45009`

- 触发限频。企业微信群机器人**每分钟最多发 20 条**。本项目正常一次运行只发 1–2 条，
  不会触发。如果你在调试时反复测试，等一分钟即可。

### 7.7 通知里的"账号异常"很多，但人工跑没问题

这是续火花本身的问题（不是通知），看 Actions 日志的具体异常。常见：
- 抖音 cookie 过期 → 重新抓 cookie
- 网络超时 → GitHub Actions 偶发问题，重跑一次

### 7.8 收到了 "续火花 Workflow 失败" 但具体原因看不到

点消息里的 `查看日志` 链接，去 Actions 页面看具体哪一步红了。可能的原因：
- `pip install` 失败 → 看 requirements
- `playwright install` 失败 → 浏览器下载超时，重跑
- main.py 启动失败 → 通常是配置文件 / 环境变量缺失

## 八、安全提示

1. **webhook URL 是密钥**：拿到的人能往你群里发任意消息（包括钓鱼）。别贴在公开 issue / 截图。
2. **GitHub Secrets 是加密的**：填进去之后再也看不到原值，只能覆盖 / 删除。这是好事。
3. **代码里不要 `print(os.getenv("FEISHU_WEBHOOK_URL"))`**：会泄露到 Actions 日志。
4. **如果 webhook 泄露**：进群机器人设置里**重置 webhook**（飞书）或**移除并重新添加机器人**（企业微信），然后更新 GitHub secret。
5. **建议为通知专门建一个群**，里面只放接收通知的人。这样即使 URL 泄露，攻击面也只是这个群。
