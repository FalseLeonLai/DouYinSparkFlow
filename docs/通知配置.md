# 运行通知配置

支持飞书自定义机器人和企业微信群机器人。任务完成 / 失败时自动推送结果。

## 启用方式

在仓库的 `Settings → Secrets and variables → Actions` 配置：

### 1. 选择渠道（Variables）

| 名称 | 值 | 说明 |
|------|----|------|
| `NOTIFY_TYPE` | `feishu` 或 `wecom` 或 `feishu,wecom` | 留空 / 不设 = 关闭通知 |

### 2. 配置 webhook（Secrets）

按选择的渠道设置：

| 名称 | 值 |
|------|----|
| `FEISHU_WEBHOOK_URL` | 飞书机器人完整 webhook URL，形如 `https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx` |
| `WECOM_WEBHOOK_URL` | 企业微信机器人完整 webhook URL，形如 `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx` |

`utils/export_github_env.py` 会自动把所有 `vars` 和 `secrets` 注入运行环境，无需改 workflow。

## 推送内容

- ✅ **完成**：账号数 / 成功好友数 / 每个账号的明细
- ⚠️ **完成（含异常）**：部分账号成功、部分异常
- ❌ **运行失败**：Python 主流程异常的完整 traceback
- ❌ **Workflow 失败**：依赖安装 / 浏览器安装 / Python 启动失败时也会推送（含 Actions 日志链接）

## 获取 webhook

### 飞书

1. 飞书群 → 设置 → 群机器人 → 添加机器人 → "自定义机器人"
2. 复制 Webhook 地址，安全设置可以选 "IP 白名单 / 关键词" 任一（或都不开）

### 企业微信

1. 企业微信群 → 右上角设置 → 群机器人 → 添加机器人
2. 复制 Webhook URL（已含 `?key=` 参数）

## 本地测试

```bash
export NOTIFY_TYPE=feishu
export FEISHU_WEBHOOK_URL=https://open.feishu.cn/...
python -m utils.notify
```

应当收到一条标题 "测试通知" 的消息。
